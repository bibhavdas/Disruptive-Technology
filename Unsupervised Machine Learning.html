Unsupervised Machine Learning

1. What is Unsupervised Machine Learning?

Imagine you have a huge bucket of mixed-up Legos, but no instruction manual telling you what to build or what the pieces are. If you decide to group them by color or size all by yourself, you are acting like an Unsupervised Machine Learning model!

The Definition: In unsupervised learning, we feed the computer unlabeled data (data that doesn't have "tags" or "right answers" attached to it). The machine's job is to analyze this data on its own and find hidden patterns, structures, and relationships without any human intervention.

The Salary vs. Spending Example: Consider a graph comparing "Salary" and "Spending". If a store feeds customer data into an unsupervised machine, it might group (or "cluster") customers together. For example, it might discover a group of people with low salary but high spending, or high salary and low spending. The machine figures out these groups entirely on its own!

2. Two Main Types of Unsupervised Learning

This branch of machine learning is typically divided into two main methodologies:

A. Association

  • What it is: This is a rule-based method used to find interesting, hidden relationships between different items.
  • The Example: The [Butter / Bread] pairing is a classic example! If a machine analyzes supermarket receipts, it might notice a pattern: If a customer buys bread, they are highly likely to buy butter. The store can use this "association" to put butter and bread next to each other in the aisle to increase sales.

B. Clustering

  • What it is: This is the method of grouping data points together. The rule is simple: objects inside the same group (cluster) should be very similar to each other, and objects in different groups should be very different from each other.

3. Types of Clustering

Clustering can be further broken down into specific algorithms:

  • Hierarchical Clustering: Think of this like a family tree. It groups similar data points into a "tree-like structure." The best part about this method is that you don't need to tell the computer how many clusters to make in advance; it builds the tree organically.
  • Partitional Clustering: This method divides data into strict, non-overlapping subsets. Every single piece of data belongs to exactly one group, with no overlap.

4. Real-World Application: Spotting Scams

Clustering is incredibly useful in the banking and finance sector. Imagine a system observing normal groups of transactions labeled CREDIT, BANK, and UPI.

What it means: If the machine groups normal, safe transactions into distinct clusters, anything that doesn't fit into those normal groups stands out as an "unclassified event." These weird, out-of-place events are often flagged as potential scams or fraud.

5. What is an Outlier?

Imagine a graph showing a cluster of dots. Most of the dots represent ages ranging from 18 to 20, so the average age is around 19.

However, there is one dot sitting way out there on its own: Age 30.

The Concept: Because "Age 30" is so different from the rest of the group, it will pull the mathematical average up, distorting the data. Data points that are drastically different from the rest of the crowd are called OUTLIERS. Unsupervised learning is great at catching these outliers!

Test Your Knowledge!

Here are some questions and answers to help you study and ensure you've mastered the concepts:

1. What is the main difference between the data used in supervised learning vs. unsupervised learning? Unsupervised learning uses unlabelled data (data that doesn't have "tags," categories, or correct answers attached to it). The machine has to figure it out on its own.
2. In your own words, what is the main goal of Unsupervised Machine Learning? The main goal is to analyze unlabelled data to find hidden patterns, structures, and relationships without any human intervention.
3. If a grocery store wants to know which products are frequently bought together, which unsupervised learning technique should they use: Association or Clustering? They should use Association.
4. Explain the "Bread/Butter" concept in the context of Machine Learning. This is an example of an Association rule. It means the machine identifies hidden relationships between items—like noticing that customers who buy bread are also highly likely to buy butter. Stores use these insights to stock related items together.
5. What is the core rule of Clustering regarding "similarities" inside a group versus outside a group? The core rule is to group objects so that objects inside the same cluster share the most similarities, while having less or no similarities with the objects of another cluster.
6. Which type of clustering creates a "tree-like structure" and does not require you to pre-define the number of clusters? Hierarchical Clustering.
7. Which type of clustering divides data into strict, non-overlapping sets where an object can only belong to one subset? Partitional Clustering.
8. How can clustering help a bank identify UPI or Credit Card scams? The machine groups standard, normal transactions into distinct clusters (like normal CREDIT, BANK, or UPI activity). If a transaction happens that does not fit into any of these normal groups, it becomes an "unclassified event." These abnormal events are flagged as potential scams.
9. What is an "Outlier"? An outlier is a piece of data that sits far outside the normal range of the rest of the dataset. It is an abnormality that does not fit in with the main cluster.
10. Using the previous example, explain how an outlier can affect the average of a dataset. Consider a cluster of people whose ages are tightly grouped between 18 and 20, meaning the average age is around 19. However, there is one person who is Age 30. Because this number is so much higher than the rest (an outlier), it will artificially pull the mathematical average up, making it look like the group is older than it actually is. Such data are called OUTLIERS.
