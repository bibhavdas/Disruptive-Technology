Data Analytics

Disruptive Technology Class Notes

1. What is Data Analytics and Data Analysis?

While often used interchangeably, they represent different scopes of work within data science:

  • Data Analysis: The highly specific, retroactive process of examining, cleaning, transforming, and modeling data to extract actionable information and answer specific questions about what happened in the past.
  • Data Analytics: The broader, overarching science that encompasses data analysis, machine learning, and statistical algorithms to find patterns, predict future trends, and drive overarching business strategy.
2. Role of data in Data Analytics.

Data acts as the foundational raw material for analytics. Without data, there is no historical context to analyze. Data provides the factual variables and evidence required to train machine learning models, validate hypotheses, and uncover hidden patterns that humans cannot see manually.

3. Why do we need to analyze data?

Data analysis shifts organizational operations from intuition-based decision-making to evidence-based decision-making. The primary reasons include:

  • Identifying operational bottlenecks and inefficiencies to reduce costs.
  • Predicting market trends and shifting consumer behavior.
  • Personalizing user experiences and targeted marketing.
  • Mitigating financial risks and detecting fraudulent activity.
4. Data analytics tools and their specific uses.
1. SQL (Structured Query Language): Used for extracting, querying, and managing data stored in relational databases.

2. Python & R: Programming languages used to write custom scripts for statistical analysis, machine learning, and complex data manipulation.

3. Tableau & Power BI: Business Intelligence tools strictly for data visualization. Used to build interactive dashboards for stakeholders.

4. Microsoft Excel: Best for basic data manipulation, ad-hoc calculations, and simple spreadsheet analysis.

5. Apache Spark / Hadoop: Distributed computing frameworks designed specifically for processing "Big Data" that standard machines cannot handle.
5. What is a Feature?

In machine learning, a Feature is an individual, independent, measurable property or characteristic of a phenomenon being observed. In a structured dataset (like a spreadsheet), features are represented by the columns.

Example: If an algorithm is designed to predict a car's top speed, the features might include "Horsepower," "Weight," and "Aerodynamic Drag."

6. What is Data Transformation?

Data Transformation is the process of converting data from one format, structure, or scale to another. This is done so the data can be integrated into a single destination database (like a data warehouse) or fed into an algorithm successfully.

7. Need of Data Transformation.

Raw data is rarely collected in a state that algorithms can immediately process. We transform data to:

  • Ensure compatibility across different systems and schemas.
  • Normalize numerical values (e.g., scaling all prices between 0 and 1) so algorithms are not mathematically biased by larger numbers.
  • Standardize formats (e.g., converting all text to lowercase, formatting dates uniformly).
8. What is Feature Engineering?

Feature Engineering is the process of applying domain knowledge to create, extract, or select new variables (features) from raw data that help machine learning algorithms perform more accurately.

Example: If a dataset contains the raw feature "Transaction Timestamp", feature engineering could extract a new feature called "Is_Weekend", which might be highly relevant for predicting retail sales spikes.

9. Data Analytics in various industries (Healthcare, Agriculture, FMCG).
  • Healthcare: Forecasting patient readmission rates, utilizing computer vision to detect anomalies in MRI scans, and analyzing genetic data to accelerate personalized drug discovery.
  • Agriculture: Deploying IoT soil sensors for precision irrigation, analyzing historical weather data to optimize planting schedules, and predicting crop yields.
  • FMCG (Fast-Moving Consumer Goods): Optimizing global supply chains, forecasting inventory demand to prevent stock-outs, and conducting "Market Basket Analysis" to see which products are frequently purchased together.
10. What is Data Pre-processing and its need?

Data pre-processing is the critical initial phase where raw data is cleaned, filtered, and organized before being modeled. It involves handling missing values, removing duplicate entries, and identifying extreme outliers.

The Need: Machine learning strictly adheres to the rule of "Garbage In, Garbage Out" (GIGO). If a model is trained on flawed, incomplete, or noisy data, its outputs will be fundamentally incorrect. Pre-processing is required to establish data integrity.
